# Final Project
**WEB-115 | Gardner, Justice**

## Overview

The WEB-115 Final Project is your chance to pursue an idea that you've always wanted to explore. You will come up with an original idea. Then, you will design, propose, and ship a complete web app using vanilla HTML, CSS, and JavaScript. This project will test your skills and allow you to apply your creativity to an original project. Projects will be developed in a `WEB-115_FinalProject_LastName` Repository.

---

## Requirements

### The Proposal

While brainstorming, you will complete a Proposal that must be approved before you begin. This Proposal will be written in a text file called `README.md` in your repo.

The proposal does not need to be as comprehensive as the Software Specification assignment but should give a good impression of the scope of the project, and how you plan to implement it. Keep in mind that you only have ~2 weeks (approx. 8-9 class days) to complete it!

### Daily Commits

You must make a daily commit **EACH DAY** by 2:15pm during the Final Project timespan. No work may be done outside of 4th period. No commits may be made outside of 4th period.

### Technical Requirements

You must incorporate:

- **If Statements and Loops**
  - Must drive real behavior in your app, not just put in for testing or to "check a box".
- **Event Listeners**
  - Enables meaningful user interaction like clicks, inputs, keyboard inputs, and custom events that change various things in your app.
- **DOM Element Creation**
  - Dynamically create and insert elements in your page using `.createElement()` and `.appendChild()`
- **Classes and Subclasses**
  - Classes and subclasses (using `extends` and `super()`) model realistic concepts and entities in your app.

## DLC (Additional Requirements)

In addition to the above requirements, you must choose to explore and incorporate **at least one topic** that we did not learn in class. If you would like to challenge yourself and create something truly impressive, feel free to incorporate more than one of these.

In your Proposal, you must name your choice(s) and explain how each will be used in your project.

| Topic | Description |
|---|---|
| **JSON & Local Storage or IndexedDB API** | Store and retrieve structured data in the browser using `localStorage` and `JSON.parse` / `JSON.stringify`. Great for apps that need to save state between sessions. |
| **Fetch & Public APIs** | Utilize at least one web API. Pull in live data using the Fetch API and display it dynamically in your app. |
| **[ADVANCED] Real-time communication using WebSockets or Socket.io** | Use WebSockets or Socket.io for real-time communication in your app. |
| **[ADVANCED] Backend Solution (Postgres, Node.js, Django)** | Use SQL and a back-end server to store and display persistent data. |
| **HTML Canvas** | Use one or more HTML Canvas elements to render shapes and draw lines. |
| **Something else?** | Have a topic in mind that isn't listed? Propose it in your `README.md`. If it fits in and extends your learning, it will be considered. |
